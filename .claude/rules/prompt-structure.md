# Task Structure

When approaching an implementation task, structure the work as:

1. **Context** — what area of the codebase, what the user goal is
2. **Spike** — verify imports resolve, function signatures match, external services reachable. If any fails, STOP and report.
3. **Task** — specific file paths and function signatures to modify
4. **Tests** — specific test scenarios
5. **Exit criteria** — what must pass for this to be done
